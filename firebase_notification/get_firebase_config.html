How to Get Your Firebase Configuration

To use Firebase Cloud Messaging, you need to provide your Firebase project's configuration details. Here's how to find them:

  1. Go to the Firebase Console and log in with your Google account.
  2. Select your Firebase project. If you don't have one, you'll need to create a new project.
  3. In the left sidebar, navigate to Project settings (the gear icon next to "Project Overview").
  4. In the "General" tab, scroll down to the "Your apps" section.
  5. If you haven't added a web app yet, click the Web () icon to add one. You'll be prompted to give it a nickname (you can choose any name). Click "Register app".
  6. Once you've added a web app (or if you already had one), click on the web app in the "Your apps" section.
  7. A configuration snippet will appear. Look for the firebaseConfig object. This object contains the following keys and values that you need to copy into the respective fields on the main page:
    • apiKey: Your API key.
    • authDomain: Your project's Auth domain.
    • projectId: Your unique project ID.
    • storageBucket: Your project's Cloud Storage bucket (may be empty).
    • messagingSenderId: Your project's Messaging Sender ID.
    • appId: Your web app's unique App ID.
    • measurementId: Your project's Measurement ID (usually starts with "G-").
  8. VAPID Key for Web Push (Voluntary Application Server Identification):
    • In the left sidebar of the Firebase Console, go to Project settings.
    • Navigate to the Cloud Messaging tab.
    • Scroll down to the "Web push certificates" section.
    • You'll see a "Default web push certificate". If a key pair hasn't been generated yet, click the "Generate Key Pair" button.
    • Copy the string under "Public key" and paste it into the VAPID Key field on the main page.
  9. Once you have copied all the necessary configuration details, navigate back to the main testing page and paste the values into the corresponding input fields. Then, click "Continue To Testing".
